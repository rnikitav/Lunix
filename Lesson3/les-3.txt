Удалить file1. Что стало с остальными созданными файлами? Попробовать вывести их на экран.


    а) mkdir ./Desktop/lesson3/                 Создаем папку на рабочем столе

    б) cd ./Desktop/lesson3/                    Переходим в папку урока 

    в) echo 'Привет Мир!' > file1.txt           Создаем файл1 с текстом Привет Мир!

        или cp file1.txt file2.txt              записываем(перенаправляем) в file2.txt

    д) ln -s file1.txt file3.txt                Создаем символическую ссылку file3.txt
                                                на file1.txt
    е) ln file2.txt file4.txt                   Создаем жесткую ссылку file4.txt 
                                                на file2.txt

    ё) ls -lai                                  i-node у символьной ссылки будет разный
                                                с файлом на который ссылка. А i-node у 
                                                у жесткой ссылки будет одинаковый с файлом
                                                на который она ссылается.

    ж) rm file1.txt                             Удаляем файл file1.txt
    
    з) ls -lai                                  file2.txt file3.txt file4.txt            
                                                Остались. Но теперь file3 подсвечивается 
                                                красным цветом, т.е если попытаться его открыть
                                                то он выдаст ошибку, потому что удален файл 
                                                на который он ссылается

    и) cat file2.txt file3.txt file4.txt        Выведет следующую информацию:
                                                Привет Мир!
                                                cat: file3.txt: No such file or directory
                                                Привет Мир!              

к) echo $?                                      Выведет 1   -->   Значит произошла ошибка
                                                Не все команды выполнились

3. Создать два произвольных файла. Первому присвоить права на чтение, запись для владельца и группы, 
    только на чтение для всех. Второму присвоить права на чтение, запись только для владельца. 
    Сделать это в численном и символьном виде.
    
    
    а) touch file1 file2                        Создаем два файла

    б) chmod 664 file1                          Права на чтение и запись для владельца и групп
       ||                                       только чтение для всех в численном       
       chmod ug+rw,o+r file1                    То же самое в символьном

    в) chmod 600 file2                          Права на чтение и запись для владельца
        ||
       chmod u=rw,g=-,o=- file2                 В символьном 


4. Создать пользователя, обладающего возможностью выполнять действия от имени суперпользователя.



    а) sudo adduser test                        Создаем юзера test

    б) sudo usermod -aG sudo test               Присоединяем его к группе суперпользователя sudo

    в) su test                                  Переключаемся на этого пользователя

    г) sudo whoami                              Проверяем чтобы вышло root
     
    е) sudo deluser test sudo                   Удаляем права суперппользователя

    ё) su test                                  переключаемся на него и Проверяем

    ж) su whoami                                test is not in the sudoers file.

5. * Создать группу developer, несколько пользователей, входящих в эту группу. 
     Создать директорию для совместной работы. Сделать так, чтобы созданные одними 
     пользователями файлы могли изменять другие пользователи этой группы.    



    а) sudo groupadd developer                  Создаем группу developer
        можем задать GID (group ID)
        sudo groupadd -g 1006 group6

    б) cat /etc/group | grep develop             Проверяем создалась ли



    в) sudo useradd -m -G developer -s /bin/bash ivanov     Создаем двух пользователей
       sudo useradd -m -G developer -s /bin/bash petrov     входящие в эту группу

    г) sudo passwd ivanov                       Устанавливаем для них пароль, т.к 
       sudo passwd petrov                       По умолчанию 
                                                для новых учетных записей без пароля 
                                                заблокирован вход в систему
     
    е) mkdir ./Desktop/develop                  Создаем новую папку для нашей группы

    ё) sudo chgrp -R developer 
       ./Desktop/Develop/                       Меняем папке Develop группу пользователей
                                                на developer

    ж) sudo chmod -R 2775                       Настройка прав доступа для папки Develop
       ./Desktop/Develop/                       -R разрешает проведение рекурсивных 
                                                    операций в подкаталоги
                                                
                                                
                                                2 - тут выполнаяет : новые файлы всегда будут 
                                                получать ту группу, 
                                                которая стоит у папки, в которой они находятся;    



    

6. Создать в директории для совместной работы поддиректорию для обмена файлами, 
   но чтобы удалять файлы могли только их создатели



    а) mkdir ShareDir                           Создаем директорию от рута и даем права

    б) sudo chmod -R 1775 ShareDir/             Из рута даем права на папку
                                                Теперь при создание файла пользователем
                                                его u - будет он сам и группа его имени
                                                Поэтому другой пользователь может читать файл,
                                                но удалить его только создатель может 
                                                Тут первая цифра (1) устанавливает sticky-битом
                                                т.е удалять может только владелец или суперползователь 

    

    Или можно сделать 
    sudo chmod -R 775 ShareDir/

    и потом добавить 
    sudo +t ShareDir/
